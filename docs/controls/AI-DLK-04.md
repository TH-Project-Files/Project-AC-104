# AI-DLK-04: Utilize AI Network Baseline Monitoring to detect anomalous network traffic patterns and alert on them

**Project:** Argus Centurion (AC-104)  
**Category:** Layer 2: Data Link  
**Implementation Group:** IG 3  
**Risk Level:** 3-Low  
**Framework Mappings:** CIS v8: `12.1, 13.4, 13.5` | NIST CSF: `DE.CM, DE.AE`

## Control Details
Detailed Description:
Implement Artificial Intelligence and Machine Learning (AI/ML) based network baseline monitoring tools to establish a profile of normal network traffic behavior. These tools continuously analyze data flows, protocol usage, and communication patterns, instantly alerting on deviations that signify unauthorized or malicious activity.

Why AI Compounds Risk:
Traditional, static threshold-based network monitoring is insufficient against AI-powered threats. Adversaries use AI to dynamically alter their attack signatures, mimic legitimate user traffic, and establish "low-and-slow" command and control (C2) channels or data exfiltration paths. Only an AI-driven defensive baseline can detect the subtle, evolving anomalies generated by AI-automated network infiltration and evasion tactics.

Examples:
1. Deploy Network Detection and Response (NDR) solutions that use unsupervised machine learning to dynamically baseline expected behavior for critical subnets and endpoints.
2. Configure automated alerts for zero-day anomalous behaviors, such as unexpected outbound data transfers (potential AI-driven data exfiltration) or unusual peer-to-peer communications.
3. Integrate AI-driven network telemetry with your SIEM/SOAR to automatically isolate endpoints demonstrating severe deviations from their learned baseline.
